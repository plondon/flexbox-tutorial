Lesson #3: flex-wrap

With flexbox we can choose to let child elements overflow their parent container, or wrap around the container's boundry. Notice that our .parent element's width is relative to the iframe, but it's willing to adjust its width based on the width of its children. This is pretty cool, and much more intuative than the standard display:box;, float:left way of the past. To understand how cool it is, try to simulate the same effect without flexbox.

tl;dr
  • flex-wrap is cool
  • the parents width will adjust to contain its children
  • the default attribute is nowrap
instructions

Experiment with the following attributes for flex-wrap. Then try to achieve the nowrap effect without display: flex;

.parent {
  ...
  flex-wrap: wrap | wrap-reverse | nowrap;
}

Please note that I've added flex:1; to the .child elements. So far we've only talked about parent properties. Children are also very important, but for now just know that flex:1; is sort of default for initializing the flexbox child.
